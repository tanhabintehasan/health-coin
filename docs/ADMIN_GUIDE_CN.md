## 1. 系统概述

HealthCoin 平台是一个基于健康理念的社交电商系统，核心特点：

- **三币体系**：健康币、互助健康币、万能健康币
- **两级分销**：直推（L1）+ 间推（L2）返佣
- **区域代理**：根据会员等级享受辖区消费返佣
- **商城抵扣**：健康币可用于抵扣商城商品
- **自动升级**：根据累计互助币自动晋升会员等级

### 1.1 管理后台入口

访问 `/admin` 路径进入管理后台，使用具有管理员权限的账号登录。

后台主要菜单：

| 菜单 | 功能说明 |
|------|---------|
| 数据概览 | 平台整体数据统计 |
| 用户管理 | 查看、编辑用户信息和等级 |
| 商户审核 | 审核入驻商家 |
| 商品审核 | 审核商家发布的商品 |
| 会员等级 | 配置会员等级门槛和返佣比例 |
| 佣金设置 | 平台佣金、提现手续费、商户抽成 |
| 系统设置 | 健康币返佣比例、支付开关等 |
| 订单监控 | 查看所有订单状态 |
| 提现审核 | 审核用户提现申请 |

## 2. 三币体系详解

### 2.1 健康币（Health Coin / HC）

- **用途**：可在商城购买商品时抵扣现金
- **获取方式**：消费后自动获得，数量 = 获得的互助币 × 倍数（默认2倍）
- **特点**：系统自动转换，不可直接配置

### 2.2 互助健康币（Mutual Health Coin / MHC）

- **用途**：可提现（扣除手续费后）
- **获取方式**：
  - 自己消费 × 50%
  - 直推会员消费 × 25%
  - 间推会员消费 × 10%
- **晋升依据**：累计获得的互助币决定会员等级

### 2.3 万能健康币（Universal Health Coin / UHC）

- **用途**：可在商城支付（无抵扣优惠）
- **获取方式**：
  - 自己消费 × 20%
  - 直推会员消费 × 10%
  - **辖区会员消费 × m%**（m值根据等级不同）

### 2.4 三币关系图

```
用户消费
    │
    ├─→ 互助币 (50%给自己, 25%给直推, 10%给间推)
    │       │
    │       └─→ 健康币 = 互助币 × 倍数(默认2倍)
    │
    └─→ 万能币 (20%给自己, 10%给直推, m%给辖区代理)
```

## 3. 会员等级与晋升

### 3.1 六个等级

| 等级 | 名称 | 晋升门槛（累计互助币） | 辖区返佣比例 m% |
|------|------|----------------------|----------------|
| L1 | 普通会员 | 0 | 0% |
| L2 | 健康大使 | 1,000 | 0% |
| L3 | 社区代理 | 5,000 | 20% |
| L4 | 县级代理 | 20,000 | 15% |
| L5 | 市级代理 | 50,000 | 10% |
| L6 | 省级代理 | 100,000 | 5% |

> 注意：门槛值在数据库中以"分"为单位存储，显示时会除以100。

### 3.2 自动升级机制

- 系统**每小时**自动检查一次所有用户
- 当用户的 `totalMutualCoinsEarned`（累计获得互助币）≥ 当前等级的 `minCoins` 时，自动晋升到下一级
- 升级是单向的，不会降级
- 管理员也可在用户管理页面手动调整用户等级

### 3.3 如何修改等级配置

**路径**：后台 → 会员等级

可修改内容：
- **等级名称**：如将"健康大使"改为其他名称
- **升级门槛**：修改晋升所需的累计互助币数量
- **辖区返佣比例 (m%)**：修改该等级对应的辖区万能币返佣比例

**操作步骤**：
1. 进入「会员等级」页面
2. 点击对应等级右侧的「编辑」按钮
3. 修改门槛值或返佣比例
4. 点击保存

> ⚠️ 警告：修改门槛后，已满足新门槛的用户将在下次自动升级任务（每小时）时升级。

## 4. 佣金计算规则

### 4.1 互助币返佣（三级之和）

| 受益人 | 比例 | 说明 |
|--------|------|------|
| 消费者自己 | 50% | 自己消费获得 |
| 直推上级 (L1) | 25% | 直接推荐人获得 |
| 间推上级 (L2) | 10% | 推荐人的推荐人获得 |

**示例**：用户A消费了 ¥100
- A获得互助币：100 × 50% = ¥50
- A的直推人获得：100 × 25% = ¥25
- A的间推人获得：100 × 10% = ¥10

### 4.2 健康币转换

- 健康币 = 消费者获得的互助币 × 倍数
- 默认倍数：**2倍**
- 以上示例中，A获得健康币：50 × 2 = ¥100

### 4.3 万能币返佣

| 受益人 | 比例 | 说明 |
|--------|------|------|
| 消费者自己 | 20% | 自己消费获得 |
| 直推上级 (L1) | 10% | 直接推荐人获得 |
| 辖区代理 | m% | 根据消费者所在辖区和代理等级决定 |

**辖区返佣说明**：
- 只有等级 ≥ L3（社区代理）的用户才能获得辖区返佣
- 当某辖区内的会员消费时，系统按该消费者的等级对应的 `regionalCoinRate` 计算辖区返佣池
- 返佣池平均分配给该辖区内所有活跃的代理（L3及以上）

### 4.4 平台佣金与结算

- **平台佣金**：订单完成后，平台抽取的佣金比例（默认5%）
- **商户默认佣金**：新入驻商户的默认平台抽成（默认5%）
- **提现手续费**：用户提现时平台扣除的手续费（默认5%）

## 5. 后台配置指南

### 5.1 系统设置 → 健康币返佣比例

**路径**：后台 → 系统设置 → Coin Reward Rates

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| Mutual Coin — Own Spend Rate | 0.5 | 自己消费获得互助币比例 |
| Mutual Coin — L1 Referral Rate | 0.25 | 直推返佣比例 |
| Mutual Coin — L2 Referral Rate | 0.1 | 间推返佣比例 |
| Health Coin Multiplier | 2.0 | 健康币 = 互助币 × 该倍数 |
| Universal Coin — Own Spend Rate | 0.2 | 自己消费获得万能币比例 |
| Universal Coin — L1 Referral Rate | 0.1 | 直推万能币返佣比例 |
| Default Mall Coin Offset Rate | 0.0 | 商城默认健康币抵扣比例 |

> ⚠️ **重要提示**：互助币三档比例之和（自己+直推+间推）建议不超过100%，否则平台可能亏损。系统会在保存时检测并发出警告。

### 5.2 佣金设置 → 平台佣金

**路径**：后台 → 佣金设置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 平台佣金比例 | 5% | 每笔订单平台抽取比例 |
| 提现手续费比例 | 5% | 用户提现手续费 |
| 商户默认佣金比例 | 5% | 新商户默认平台抽成 |

### 5.3 佣金设置 → 会员级别辖区返佣比例

**路径**：后台 → 佣金设置 → 会员级别辖区返佣比例 (m%)

这里显示每个会员等级对应的辖区返佣比例，可直接编辑：

| 等级 | 默认比例 | 建议范围 |
|------|---------|---------|
| 普通会员/健康大使 | 0% | 保持0 |
| 社区代理 | 20% | 15-25% |
| 县级代理 | 15% | 10-20% |
| 市级代理 | 10% | 5-15% |
| 省级代理 | 5% | 3-10% |

> 高级别代理比例较低，是因为高级别代理管辖范围更大，总消费量更多。

### 5.4 支付开关

**路径**：后台 → 系统设置 → Payment Providers

| 开关 | 说明 |
|------|------|
| Fuiou Payment Enabled | 富友聚合支付（微信/支付宝/银行卡） |
| LCSW Payment Enabled | 扫呗支付 |
| Coin Payment Enabled | 健康币/互助币/万能币支付 |

## 6. 商城抵扣功能

### 6.1 抵扣规则

- 每个商品可设置「健康币抵扣比例」（0% ~ 100%）
- 用户在商品详情页可看到「健康币可抵 XX%」的标识
- 支付时系统自动计算抵扣金额

**抵扣计算方式**：
```
抵扣金额 = 订单总额 × 商品抵扣比例
现金支付 = 订单总额 - 抵扣金额
```

**示例**：
- 订单总额 ¥100，抵扣比例 30%
- 自动从用户健康币账户扣除 ¥30
- 用户只需现金支付 ¥70

### 6.2 默认抵扣比例

- 全局默认值可在「系统设置」中配置 `Default Mall Coin Offset Rate`
- 商户发布商品时可自行设置该商品的抵扣比例
- 如果商户未设置，则使用全局默认值

### 6.3 健康币支付的特殊规则

- 选择「健康币支付」时，如果商品有抵扣比例，用户实际只需支付抵扣后的金额
- 选择「互助币/万能币支付」时，需支付全额（无抵扣优惠）
- 选择「现金支付」时，健康币自动抵扣，现金支付剩余部分

## 8. 常见问题 FAQ

### Q1: 为什么用户消费后没有获得健康币？

**可能原因**：
1. 订单尚未完成支付（返佣在支付确认后触发）
2. `payment_coin_enabled` 设置为 false
3. 返佣处理异步失败（可查看服务器日志）

**排查方法**：
- 检查订单状态是否为 PAID
- 检查 `coin-rewards.service.ts` 日志
- 检查 `wallet_transactions` 表是否有对应记录

### Q2: 辖区返佣为什么没有发放？

**可能原因**：
1. 消费用户所在辖区没有代理（L3及以上）
2. `regionalCoinRate` 为 0
3. 消费用户自己就是该辖区唯一的代理（排除自己）

**排查方法**：
- 确认消费用户有 `regionId`
- 确认辖区内有 `membershipLevel >= 3` 的活跃用户

### Q3: 会员等级为什么不自动升级？

**可能原因**：
1. 自动升级任务是每小时运行一次，不是实时的
2. `totalMutualCoinsEarned` 字段未更新（返佣失败）
3. 数据库中 `minCoins` 值不正确

**排查方法**：
- 手动调用 `POST /admin/membership/tiers/seed` 重新初始化等级
- 检查 `membership.service.ts` 的 cron 任务是否正常运行

### Q4: 商城抵扣比例为什么不生效？

**可能原因**：
1. 商品未设置 `coinOffsetRate`
2. 用户健康币余额不足抵扣金额
3. 用户选择了互助币/万能币支付（这两种支付方式不支持抵扣优惠）

### Q5: 如何修改健康币与互助币的兑换比例？

**方法**：进入后台 → 系统设置 → Coin Reward Rates → 修改 `Health Coin Multiplier`
- 默认值为 2.0，表示 1 互助币 = 2 健康币
- 修改为 3.0 后，1 互助币 = 3 健康币

### Q6: 平台会亏损吗？

**风险点**：
- 如果互助币三档返佣之和 > 100%，每消费一笔平台就亏损
- 建议保持：50% + 25% + 10% = 85% ≤ 100%
- 系统会在保存配置时检测并警告
