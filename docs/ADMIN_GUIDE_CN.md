## 一、登录与访问

### 访问地址
```
http://<服务器IP>/login
```

### 管理员账号
- 使用后台管理员账号登录（非普通用户账号）。
- 若提示无权访问，请联系技术人员确认该账号已加入 `admin_users` 表且 `isActive = true`。

### 进入后台
登录成功后，访问：
```
http://<服务器IP>/portal/admin
```

## 二、数据概览

**路径：** 左侧菜单 → 数据概览

### 功能说明
- 查看平台核心运营数据：总用户数、总商户数、总订单数、总交易额。
- 查看今日新增用户、今日订单、今日交易额。
- 查看三币（互助健康币、健康币、万能健康币）发行总量。

### 使用建议
- 每日开盘前查看数据概览，掌握平台整体健康度。

## 三、用户管理

**路径：** 左侧菜单 → 用户管理

### 功能说明
- **查看用户列表：** 显示所有注册用户的基本信息（手机号、昵称、会员级别、推荐码、所属辖区）。
- **搜索/筛选：** 可按手机号、昵称、会员级别搜索。
- **编辑用户：** 点击"编辑"可修改用户昵称、手机号、会员级别、所属辖区、头像。
- **新建用户：** 管理员可手动创建新用户，系统会自动生成推荐码。
- **重置密码：** 可在编辑弹窗中为用户设置新密码。

### 注意事项
- 修改用户会员级别后，系统不会自动重新计算该用户的历史奖励，仅影响后续订单。
- 删除用户需谨慎，关联的订单、钱包记录不会被级联删除。

## 四、商户审核

**路径：** 左侧菜单 → 商户审核

### 功能说明
- **查看入驻申请：** 列出所有提交入驻申请的商户。
- **审核操作：**
  - **通过：** 商户状态变为 `APPROVED`，可上架商品。
  - **拒绝：** 需填写拒绝原因，商户可重新提交。
- **查看详情：** 点击商户名称查看营业执照、法人信息、联系方式等。

### 审核标准建议
- 营业执照真实有效。
- 经营类目与平台定位（健康、养生、医疗相关）相符。
- 联系人信息完整。

## 五、商品审核

**路径：** 左侧菜单 → 商品审核

### 功能说明
- **待审核列表：** 显示商户提交的所有待上架商品。
- **批量审核：** 可批量通过或批量拒绝。
- **查看详情：** 查看商品图片、价格、库存、健康币抵扣比例、运费模板等。

### 自动上架开关
在【系统设置】中可开启/关闭 `"product_review_required"`：
- `true`（默认）：所有新商品需管理员人工审核后上架。
- `false`：商户发布商品后自动上架，无需审核。

## 六、分类管理

**路径：** 左侧菜单 → 分类管理

### 功能说明
- **一级/二级分类：** 支持两级商品分类体系。

## 七、订单监控

**路径：** 左侧菜单 → 订单监控

### 功能说明
- **查看所有订单：** 包括待付款、已付款、待发货、已发货、已完成、已取消、退款中。
- **搜索订单：** 按订单号、用户手机号、商户名称搜索。
- **强制修改状态：** 管理员可强制修改订单状态（仅用于异常处理）。
- **查看详情：** 订单商品明细、支付信息、物流信息、三币发放记录。

### 订单状态流转
```
待付款 → 已付款 → 待发货 → 已发货 → 已完成
   ↓        ↓         ↓
 已取消   退款中    退款中
```

## 八、提现审核

**路径：** 左侧菜单 → 提现审核

### 功能说明
- **待审核列表：** 用户提交的提现申请。
- **审核操作：**
  - **通过：** 标记为已打款，用户钱包扣除相应金额。
  - **拒绝：** 需填写原因，金额原路退回用户钱包。

### 提现手续费
平台默认扣除 **5%** 手续费（可在【系统设置】→ `withdrawal_commission_rate` 中调整）。

## 九、会员等级（重点）

**路径：** 左侧菜单 → 会员等级

### 功能说明
会员等级决定用户的升级门槛和辖区返佣比例（m%）。

### 默认六级体系
| 级别 | 名称 | 升级门槛（互助健康币） | 辖区返佣比例 m% |
|---|---|---|---|
| L1 | 普通会员 | 0 | 0% |
| L2 | 健康大使 | 1,000 | 0% |
| L3 | 社区代理 | 5,000 | 20% |
| L4 | 县级代理 | 20,000 | 15% |
| L5 | 市级代理 | 50,000 | 10% |
| L6 | 省级代理 | 100,000 | 5% |

### 核心操作

#### 1. 恢复默认等级
- 若页面显示"暂无数据"，点击右上角 **"恢复默认值"** 按钮，一键生成上述6个默认等级。
- 已有数据时，此按钮会自动隐藏。

#### 2. 编辑等级
- 点击每行右侧的 **"编辑"** 按钮。
- 可修改：
  - **等级名称**：如"健康大使"、"社区代理"。
  - **升级门槛**：用户累计获得多少互助健康币可自动升级到此级别（单位：个）。
  - **辖区返佣比例（m%）**：万能健康币计算中"所属辖区会员消费 × m%"的 m 值。
  - **描述**：该等级的权益说明。
- 点击 **"保存更改"** 后立即生效。

#### 3. 新增等级
- 点击右上角 **"新增等级"** 按钮。
- 填写：
  - **级别数字**：必须是唯一正整数，数字越大代表等级越高。
  - **等级名称、升级门槛、辖区返佣比例、描述**。
- 点击 **"创建等级"**。

#### 4. 删除等级
- 点击每行右侧的 **"删除"** 按钮。
- **限制：** 如果当前有用户属于该等级，则无法删除。需先将这些用户调整到其他等级。

### 万能健康币计算公式
```
万能健康币 = 自己消费 × 20% + 直推会员消费 × 10% + 所属辖区会员消费 × m%
```
其中 **m%** 根据会员级别从本页面设置。

### 自动升级
系统每小时自动扫描一次，若用户累计互助健康币达到更高级别的门槛，则自动升级。

## 十一、佣金设置（重点）

**路径：** 左侧菜单 → 佣金设置

### 功能说明
集中管理所有与佣金、手续费、返佣比例相关的参数。

### 平台佣金
| 参数 | 说明 | 默认值 |
|---|---|---|
| 平台佣金比例 | 每笔订单平台抽取的佣金比例 | 5% |
| 提现手续费比例 | 用户提现时平台扣除的手续费 | 5% |
| 商户默认佣金比例 | 新入驻商户的默认平台抽成比例 | 5% |

### 会员级别辖区返佣比例
- 本区域以表格形式展示所有会员级别的 **m%** 值。
- 可直接点击 **"编辑"** 修改某级别的 m%。
- 与【会员等级】页面数据实时同步。

### 保存
修改后点击 **"保存平台佣金设置"**，所有参数立即生效。

## 十四、系统设置

**路径：** 左侧菜单 → 系统设置

### 功能说明
集中配置平台所有全局参数。

### 三币奖励比例
| 参数 | 说明 | 默认值 |
|---|---|---|
| 互助健康币—自购比例 | 买家消费获得互助健康币的比例 | 50% |
| 互助健康币—直推比例 | 一级推荐人获得的互助健康币比例 | 25% |
| 互助健康币—间推比例 | 二级推荐人获得的互助健康币比例 | 10% |
| 健康币倍数 | 每获得1个互助健康币，同时获得多少健康币 | 2倍 |
| 万能健康币—自购比例 | 买家消费获得万能健康币的比例 | 20% |
| 万能健康币—直推比例 | 一级推荐人获得的万能健康币比例 | 10% |

### 支付开关
- 富友支付、利楚扫呗、微信支付、支付宝支付、健康币支付。
- 设置为 `1` 开启，`0` 关闭。

### 短信配置
- 短信服务商、模板、签名、账号密码。
- OTP有效期、重发冷却时间、每小时限制次数。

### 微信小程序
- AppID、AppSecret（用于微信登录）。

### 保存
修改后点击 **"保存设置"**。

## 十五、常见问题

### Q1：会员等级页面显示"暂无数据"怎么办？
**A：** 点击页面右上角的 **"恢复默认值"** 按钮，系统会自动创建6个默认会员等级。若按钮不存在，说明前端版本未更新，请联系技术人员部署最新代码。

### Q2：修改了佣金比例，为什么历史订单没有变化？
**A：** 佣金/奖励比例仅影响**新产生的订单**，历史订单的奖励金额在结算时已固定，不会回溯调整。

### Q3：新增了一个会员等级（如 L7），现有用户会自动升级吗？
**A：** 会。系统每小时自动扫描一次，只要用户的累计互助健康币达到新等级的门槛，就会自动升级。

### Q4：能否删除某个默认等级（如"普通会员"）？
**A：** 可以，但前提是该等级下没有任何用户。建议保留 L1（普通会员），因为新注册用户默认就是 L1。

### Q5：页面修改后没有生效？
**A：**
1. 确认已在服务器执行了构建命令（`npm run build:api` 和 `npm run build:web`）。
2. 确认 PM2 服务已重启（`pm2 restart healthcoin-api` 和 `pm2 restart healthcoin-proxy`）。
3. 浏览器按 **Ctrl + F5** 强制刷新，清除缓存。

### Q6：如何查看后台接口是否正常工作？
**A：** 在服务器上执行以下命令测试：
```powershell
cd C:\healthcoin\apps\api
curl -H "Authorization: Bearer <你的管理员Token>" http://localhost:3000/api/v1/admin/membership/tiers
```
若返回 JSON 数组则正常；若返回 500 错误，请检查 API 构建日志。

## 附：快速更新命令（RDP服务器）

在服务器 PowerShell 中执行：

```powershell
cd C:\healthcoin

# 拉取最新代码
git pull

# 安装依赖
npm install

# 构建后端
cd C:\healthcoin\apps\api
npm run build

# 构建前端
cd C:\healthcoin\apps\web
npm run build

# 重启服务
pm2 restart healthcoin-api
pm2 restart healthcoin-proxy
pm2 save
```

或者使用一键脚本：
```powershell
C:\healthcoin\scripts\quick-update.ps1
```
